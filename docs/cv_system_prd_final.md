# 简历智能润色系统 - 任务确认单 (终版)

## 1. 核心目标
构建一个本地智能化的简历修改与润色系统，解决使用 HTML 编写简历时直观性差、修改繁琐的问题，并结合大语言模型提供专家级的简历优化建议与自动修改能力。

## 2. 角色定位 (AI)
- 顶级简历修改专家
- 资深职业规划专家

## 3. 核心功能及交互流程

### 3.1 页面布局 (三栏式设计)
*   **左侧 (AI 交互区):**
    *   **输入区:** 用户输入目标岗位的 JD (Job Description) 或自定义的修改诉求 (如：格式、内容、模板调整等)。
    *   **反馈区:** AI (专家系统) 根据 JD 或诉求分析简历不足，提供明确的优化建议及修改方向。
    *   **历史版本管理:** 记录和管理历次修改的 HTML 代码版本。
*   **中间 (代码/编辑区):** 展示简历当前的 HTML 源码，可供用户手动修改或 AI 自动更新。
*   **右侧 (实时预览区):** 实时渲染中间的 HTML 代码，提供最终展示效果。

### 3.2 自动化修改流程
1.  **提交需求:** 用户向 AI 输入 JD 或修改诉求。
2.  **智能分析:** AI 提供分析及修改建议。
3.  **确认执行:** 用户确认修改方向。
4.  **自动修改:** AI 自动生成新的 HTML 代码并覆盖中间区域，右侧区域同步刷新。
5.  **版本保存:** 每次 AI 自动修改或用户重大更改后，系统自动保存当前 HTML 版本至历史记录。

### 3.3 初始资源
系统内置一份精美的基础 HTML 简历模板，用户可通过上述流程或手动随时修改其内容与结构。

## 4. UI/UX 及架构要求
- **系统类型:** 本地工具 (无强制登录及云端部署需求)。
- **风格设计:** 界面高级、专业。
- **全局语言:** 全局界面文字必须使用**中文**。
- **技术栈:** 
    - **前端:** 不涉及后端框架，纯静态应用或利用轻量本地服务运行。
    - **大模型接口:** 需集成 OpenAI / Google Gemini 等 API (后续需配置 API Key)。
    - **历史记录存储:** 利用浏览器 LocalStorage 或本地轻量级数据库 (如 SQLite) 进行版本保存。
